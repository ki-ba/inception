# Inception

## Description

Inception is a project that involves creating a web application using Docker / Docker compose.
The main services to be implemented are:
- Nginx: A web server that will serve the application.
- MySQL: A database server to store the application's data.
- WordPress: A content management system (CMS) that will be used to create and manage the website.

Its goal is to introduce the student to the concepts of containerization and orchestration using Docker, as well as to provide hands-on experience in setting up and managing a web application stack. The project will require the student to create Dockerfiles for each service, configure the necessary settings, and use Docker Compose to orchestrate the services together.
### Project description

**What is Docker and how is containerization different from virtualization?**

**Docker** is a platform that allows developers to create, deploy, and run applications in **containers**.
Containers are **lightweight, self-sufficient units** that can run consistently across different environments.
They encapsulate an application and its dependencies, ensuring that it runs the same way regardless of where it is deployed.
Even though both containerization and virtualization allow for the isolation of applications, they differ in how they achieve this :
virtualization involves creating a VM which abstracts the underlying hardware. Each VM includes **its own operating system**, which can lead to increased resource usage.
In contrast, containerization **shares the host operating system's kernel**, allowing for more efficient use of resources and faster startup times.

**What are docker Secrets and how do they enhance security in a Docker environment?**

**Docker Secrets** are a secure way to manage sensitive information, such as passwords, API keys, and certificates, within a Docker environment.
They allow you to store and manage secrets separately from the application code, ensuring that sensitive data is not exposed in the Dockerfiles or environment variables.
As they are encrypted, they cannot be the target of unauthorized access as an environment variable or a file in the container's filesystem.

**What is Docker Network?**

**Docker Network** is a feature of Docker that allows containers to communicate with each other and with the outside world. It provides a way to create isolated networks for containers, enabling them to communicate securely and efficiently.

**What are Docker Volumes and how do they differ from bind mounts?**

**Docker Volumes** are a way to persist data generated by and used by Docker containers. They are managed by Docker and can be easily shared between containers. Volumes are stored in a part of the host filesystem that is managed by Docker, which allows for better performance and security.
In contrast, **bind mounts** allow you to mount a file or directory from the host filesystem into a container. This can be useful for development purposes, but it can also lead to security issues if not used carefully, as it gives the container access to the host's filesystem.

## Instructions

### Requirements

Before starting the project, make sure you have:
- Docker installed,
- Docker Compose installed,
- the project files available locally,
- a valid `.env` file containing the required environment variables.

### Environment variables

The stack expects the following variables:

- `MARIADB_ROOT_PASSWORD`
- `MARIADB_DATABASE`
- `MARIADB_USER`
- `MARIADB_PASSWORD`
- `WP_ADMIN`
- `WP_ADMIN_PASS`
- `WP_ADMIN_EMAIL`
- `USER_LOGIN`
- `USER_PASSWORD`
- `USER_EMAIL`

### Launch the project

From the project root, run:

```bash
make build
```

This will build the images and start all containers in detached mode.

### Useful commands

- `make up` to start the stack without rebuilding.
- `make down` to stop and remove the containers.
- `make stop` to stop the containers.
- `make ps` to display running containers.
- `make clean` to stop the stack and remove volumes.
- `make restart` to restart the stack.
- `make re` to clean and rebuild the project.
- `make nuke` to stop everything and prune unused Docker resources.

### Access the website

Once the stack is running, the website should be available at:

```text
https://kbarru.42.fr
```

If necessary, configure your local hosts file so that the domain points to `127.0.0.1`.

### Notes

- WordPress data is stored in the `wp_data` volume.
- MariaDB data is stored in the `mariadb_data` volume.
- The containers communicate through the `netception` Docker network.
- Nginx uses a self-signed TLS certificate generated during image build.
